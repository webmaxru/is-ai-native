WebMCP · live demo

This page is an MCP server your browser exposes to AI agents

WebMCP lets a web page register tools directly on the page so that an AI agent — running in your browser, an extension, or an MCP client — can call them with your live session and permissions. This demo page registers the same repository-scanner tools used across Is AI-Native?, so you can inspect and invoke them right here.

Available tools

Two tools are registered on this page. Inspect them in your WebMCP client of choice, then call them with a GitHub repository such as facebook/react or microsoft/vscode.

scan_repository imperative

Scans a GitHub repository for AI-native development primitives and updates the visible report. Registered in JavaScript via modelContext.registerTool() with a readOnlyHint annotation.

Input
repo_url string · required
Accepts owner/repository or a full https://github.com/owner/repository URL.
scan_repository_form declarative

The same scan, exposed declaratively by annotating the on-page form below with toolname, tooldescription, toolautosubmit, and a toolparamdescription on the input. The agent fills the field and the form submits itself.

Input
repo_url string · required
Bound to the live form field — try it below.

How to test it

WebMCP is shipping behind an experimental flag and origin trial. Pick whichever client fits your demo — both detect the tools registered on whatever page is in the foreground.

1

WebMCP Model Context Tools extension

A Chrome extension that surfaces a page's WebMCP tools and bridges them to an MCP client so any compatible agent can call them.

  1. Install the WebMCP Model Context Tools extension from the Chrome Web Store.
  2. Keep this page open and pin the extension, then open its popup.
  3. Confirm scan_repository and scan_repository_form are listed for this tab.
  4. Connect the extension to your MCP client and ask the agent to scan a repository.
Open the extension →
2

Chrome DevTools WebMCP panel

Chrome DevTools includes a WebMCP panel that lists the tools on the inspected page and lets you invoke them with sample input — no extension required.

  1. Use a recent Chrome (Canary/Dev) with WebMCP enabled, then open DevTools (F12).
  2. Open the WebMCP panel (under Application, or via the » overflow / Command Menu).
  3. Select scan_repository, provide a repo_url, and run it.
  4. Watch the form fill and the result render on this page.
